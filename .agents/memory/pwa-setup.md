## Rule
PWA is configured via `vite-plugin-pwa` in vite.config.ts. Icons are in `public/icons/` (generated by `/tmp/gen-icons.mjs` pure Node.js PNG encoder — no external deps needed). Offline fallback is `public/offline.html`.

Workbox strategies:
- `/api/dashboard/*` → NetworkFirst, 5s timeout, 5min TTL
- `/api/(products|categories|customers)` → NetworkFirst, 8s timeout, 10min TTL
- Static assets (.js, .css, .woff2) → CacheFirst, 30d TTL
- Images (.png, .svg) → CacheFirst, 7d TTL

navigateFallbackAllowlist: `[/^(?!\/api\/)/]` — API routes excluded from SW intercept.

**Why:** The app is a SPA (TanStack Router client-side). navigateFallback is safe. API routes must bypass SW to always go to the Express server.

**How to apply:** If bumping cache keys (e.g. after a schema change that invalidates cached API responses), increment the cacheName version suffix (api-dashboard-v1 → v2, etc.).
